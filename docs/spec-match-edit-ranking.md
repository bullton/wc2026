## 3. 2026 世界杯积分排名规则

### 3.1 小组内同分球队排名（6 层破平局）

| 优先级 | 规则 | 说明 |
|--------|------|------|
| 1 | 胜负关系净胜球 | 同分球队直接交锋的净胜球 |
| 2 | 胜负关系进球数 | 同分球队直接交锋的总进球 |
| 3 | 总净胜球 | 小组赛所有比赛的总净胜球 |
| 4 | 总进球数 | 小组赛所有比赛的总进球 |
| 5 | 公平竞赛积分 | 黄牌-1，红牌-4，两黄变一红-3（扣分少者排前） |
| 6 | FIFA 世界排名 | 排名靠前者排前 |

### 3.2 最好的 8 个小组第三名

同样适用上述规则比较。

## 4. 32强对阵矩阵

### 4.1 数据来源

FIFA 官方发布的 32 强对阵矩阵，包含 495 种场景，描述当12个小组中哪8个小组直接晋级时，8支第三名球队的排名位置。

### 4.2 数据结构

```json
{
  "scenario_id": 1,
  "qualifier_groups": ["E", "F", "G", "H", "I", "J", "K", "L"],
  "third_places": ["3E", "3J", "3I", "3F", "3H", "3G", "3L", "3K"]
}
```

- `qualifier_groups`：直接晋级的小组（前两名晋级）
- `third_places`：8支第三名球队按实力排序（最好的在前面）

### 4.3 数据库表

```sql
CREATE TABLE IF NOT EXISTS knockout_matrix (
    id INTEGER PRIMARY KEY,
    scenario_id INTEGER,
    qualifier_groups TEXT,
    third_places TEXT
);
```

## 5. 淘汰赛对手自动生成

### 5.1 算法流程

1. 保存小组赛结果后，计算所有12个小组的排名
2. 确定直接晋级的小组（每组前两名）和12支第三名球队
3. 比较12支第三名球队，按 FIFA 规则排序选出前8名
4. 在矩阵中匹配 qualifier_groups 最相似的情况
5. 根据匹配结果更新1/16决赛中涉及第三名的比赛

### 5.2 匹配逻辑

矩阵包含494种 qualifier_groups 组合，每种组合对应8支第三名的固定排序。

当实际晋级情况匹配到某个场景时：
- `third_places[0]` → 成绩最好的第三名 → 场序79/85的主场或客队
- `third_places[1]` → 成绩第二好的第三名 → 场序74的对手
- ...

## 7. 数据流

```
用户点击卡片
    ↓
弹出编辑表单
    ↓
用户输入 → 保存
    ↓
PUT /api/matches/{id}
    ↓
更新数据库
    ↓
计算小组排名 → 选出 8 个最好第三
    ↓
匹配对阵矩阵 → 更新淘汰赛对手
    ↓
前端拉取最新数据
    ↓
重新渲染积分表（自动排名）
    ↓
更新淘汰赛对阵图
```
